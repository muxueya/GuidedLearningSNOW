Architecture Approaches

Choose the foundation for the Guided Learning tool

A

Pure SPA — Vite + React (client-only)

Claude API called directly from the browser. API key in .env. All state in Zustand + IndexedDB. Tone.js for audio. Agent tools limited to browser-safe operations.

Pros

  • Simplest setup
  • Single npm run dev
  • No backend at all

Cons

  • API key visible in browser bundle
  • Agent tools limited (no web search)
  • Streaming requires CORS proxy workaround
B

⭐ Next.js Full-Stack (recommended)

Single repo. React frontend + API routes handle all Claude calls server-side. API key never leaves the server. Agent tools run on the server (web search, file I/O). Single npm run dev starts everything.

Pros

  • API key stays secure
  • Rich agent tools (web search, notes)
  • Streaming via Server-Sent Events
  • One command to run

Cons

  • Next.js has some framework opinionation
  • Slightly heavier than bare Vite
C

Vite SPA + Separate Express API

Two separate processes: Vite dev server for frontend, Express/Fastify for the Claude API layer and tools. Most flexible and explicit separation.

Pros

  • Clear frontend/backend separation
  • Easy to swap backend later
  • Full tool execution support

Cons

  • Two processes to run
  • More boilerplate
  • No real advantage over Next.js for a personal tool
